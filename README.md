Упрощённая модель функционирования облачного сервиса. 

Входные данные: 
    1) Время действия сервиса (целое число минут)
    2) Максимальное количество клиентов, которые одновременно могут находится в сессии (целое число клиентов)
    3) Количество пользователей, которые прибавляются каждую минуту (целое число пользователей)
    Диапазон времени работы клиентов(в минутах) Например: каждый клиент может работать от 10 до 45 минут (случайное целое число в этом диапазоне)
    4) Начало диапазона (целое число минут)
    5) Конец диапазона (целое число минут)
    
Выходные данные:
    Для каждой минуты действия сервиса программа будет выводить номер этой минуты, номера клиентов в активной сессии, количество клиентов в очереди и саму очередь
    
    Например: у нас есть облачный игровой сервис. Мы хотим смоделировать его работу на 70 минут (1-ый параметр во входных данных). 
    Всего компьютеров у нас 15, поэтому максимальное количество клиентов, которые одновременно могут находится в сессии(2-ой параметр во входных данных) будет равнятся 15. 
    Каждую минуту в наш игровой сервис подключаются по 2 человека (3-ий параметр во входных данных).
    В среднем каждый пользователь сидит от 30 до 50 минут. (4-ый и 5-ый параметр во входных данных соответственно)

    Вот результат на 15-ой минуте. И в таком формате программа вывела результат на каждой из 70-ти минут.
    В данном примере на 70-ой минуте у меня получилось 110 клиентов в очереди, а последний вошедший в активной сессии под номером 30.

    ВАЖНО! Т.к. время работы каждого клиента - случайное значение в заданном диапазоне, то результаты даже с одинаковыми входными данными будут различатся, что вполне логично, ведь нельзя заранее точно знать, сколько времени будет сидеть каждый клиент.
